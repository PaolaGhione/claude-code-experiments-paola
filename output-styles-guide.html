What Are Output Styles & Which Ones Do You Have?

> WHAT IS AN OUTPUT STYLE?

An Output Style is a Claude Code configuration file that instructs the AI to format every single response in a specific, consistent way — overriding the default markdown prose that Claude would normally produce.

Think of it as a permanent formatting contract: once an output style is active, Claude stops deciding how to present its answers and instead follows the template you defined. This affects not just cosmetics — it shapes the structure, vocabulary, and medium of every reply.

> WHERE DO THEY LIVE?

Output style files are .md files stored under .claude/output-styles/ inside your project directory. They use YAML frontmatter for metadata and a markdown body for the actual formatting instructions: 

---
name: my-style          <-- unique identifier
description: one-liner  <-- shown in picker/listings
keep-coding-instructions: true   <-- OPTIONAL flag (see below)
---

Format every response as ...   <-- the actual instructions Claude follows

> THE ANATOMY OF AN OUTPUT STYLE FILE

FieldRequiredPurpose
nameYesUnique slug used to select / reference the style
descriptionYesShort description shown in listings
keep-coding-instructionsNoIf true, Claude keeps its built-in coding rules even while applying the style
body (below frontmatter)YesPlain-english instructions describing exactly how to format responses

> HOW ARE THEY ACTIVATED?

Output styles are injected into Claude's system prompt at session start via the Output Style section. The active style is whatever file is referenced in your project's .claude/settings.json (or selected in the Claude Code UI). Once active, Claude reads the body of that file as a binding instruction for every turn of the conversation.

> THE keep-coding-instructions FLAG

By default, an output style replaces the normal response behavior entirely. If your style is highly visual (like an HTML blog renderer), it could accidentally suppress Claude's built-in coding guidelines — things like "don't add unnecessary comments", "prefer edits over new files", etc.

Setting keep-coding-instructions: true in the frontmatter tells the harness to inject the coding rules alongside the style, so both apply at once. Use this whenever your style governs presentation but you still want rigorous coding behaviour.

> WHY USE OUTPUT STYLES?

  • Consistency — every answer looks the same, every time, regardless of the question
  • Automation — output can be piped directly into tools (e.g. YAML → parsed by scripts)
  • Branding / UX — demos and course material can have a distinctive visual identity
  • Cognitive focus — a strict format forces terse, structured thinking
  • Experimentation — test different communication styles without modifying prompts

Your Current Output Styles

Two output styles are currently defined in .claude/output-styles/:

1. retro-ascii-blog
File: .claude/output-styles/retro-ascii-blog.md
ACTIVE NOW keep-coding-instructions: true

Renders every Claude response as a complete, self-contained HTML page styled as a retro ASCII terminal blog. The response is raw HTML — no markdown, no prose outside the HTML block. The file is saved to disk and opened in the default browser automatically.

Key design tokens:

  • Background #0a0a0a, primary text #33ff33 (terminal green)
  • Font: Share Tech Mono (Google Fonts), fallback Courier New
  • Container: max-width 800px, green border + glow box-shadow
  • Structure: ASCII banner → meta line → h1 → content → footer with blinking cursor
  • Saves each response as a descriptive .html file and opens it in the browser

Best for: demos, course content, visual blog-style documentation

2. yaml-concise
File: .claude/output-styles/yaml-concise.md
INACTIVE

Formats every response as pure YAML — key-value pairs, hierarchical, no prose paragraphs whatsoever. Extremely terse and machine-readable.

Schema:

answer:
  summary: one-line answer
  details:
    - point one
    - point two
  caveats:
    - only if relevant

Rules: no prose, nest with indentation not headers, omit empty keys, values as short as possible, no preamble or closing remarks.

Best for: scripts that parse Claude's output, quick structured lookups, CI pipelines

> HOW TO ADD A NEW OUTPUT STYLE

  1. Create .claude/output-styles/my-new-style.md
  2. Add YAML frontmatter with name, description, and optionally keep-coding-instructions: true
  3. Write the formatting instructions in the body
  4. Select it via the Claude Code UI or reference it in .claude/settings.json
